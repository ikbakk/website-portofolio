00 / Brief

Four routes, one system.

A working engineer's personal brand needs to read as a single, deliberate voice across the portfolio, the GitHub profile, and the resume header. This document proposes four directions that all bind to the active WeChat design system so swapping between them changes only the mark, not the page.

Every direction uses the same six tokens (--bg / --surface / --fg / --muted / --border / --accent), the same Newsreader + IBM Plex Sans + JetBrains Mono stack, and the same six spacing tiers. They differ in silhouette: how much the mark carries versus how much the typography carries.

A is the safest reading. C is the most editorial. B is the most original. D is the most communicative. Pick the one that matches the way you want to be remembered.

Constraints
No em dashes. No Tailwind. No fabricated metrics. No external image CDNs. The mark must render without webfonts (system fallbacks present) and degrade gracefully in print.

Deliverables per route
Symbol (SVG) / Wordmark (HTML) / Lockup (HTML composition) / Palette swatches / Usage rules (do + don't) / Generation prompt (raster-ready).

A

Speak

A single green dot inside an empty speech bubble. The most restrained mark in the system / it reads as a quiet invitation to talk, not a logo shouting for attention.

Symbol
Lockup
[Your name] Frontend engineer

Palette

--accent#07c160
--bg#ededed
--fg#1a1a1a

Stroke uses --accent at 2.5px. Backgrounds in the lockup use --bg. The single interior dot is the only filled element.

Usage rules

  • +Use as the favicon / the dot is legible at 16px.
  • +Use as the social-card avatar (paired with the bg).
  • −Don't fill the bubble interior / emptiness is the point.
  • −Don't add a second dot, no matter how minimal it feels.

D

Dialogue

Two interlocking speech forms / one filled, one outlined / offset diagonally. References the chat-app heritage of the design system, but reads as two voices in conversation rather than a single brand shouting.

Lockup

Palette

--fg#1a1a1a
--bg#ededed

The filled bubble uses --accent. The outlined bubble uses --fg at 2px. The mark carries the system's full dialogue (color + structure), so use it sparingly on the page.

Usage rules

  • +Use as the section marker for "Selected work" / the two voices read as project / context.
  • +Use on the GitHub profile or social card / the silhouette is distinctive at 64px.

+ / Apply

Drop the mark into the portfolio.

All four marks are pure SVG. The same four files below are already inlined in the rail of frontend-engineer-portfolio.html with a live direction-switcher (data-direction on body). The copy here is the integration receipt.

HTML

<!-- rail-brand -->
<a href="#page-01" class="rail-mark" data-direction="a" aria-label="Home">
  <svg class="mark mark-a" ...>...</svg>
  <svg class="mark mark-b" ...>...</svg>
  <svg class="mark mark-c" ...>...</svg>
  <svg class="mark mark-d" ...>...</svg>
</a>

CSS

.mark { display: none; width: 28px; height: 28px; }
body[data-direction="a"] .mark-a { display: block; }
body[data-direction="b"] .mark-b { display: block; }
body[data-direction="c"] .mark-c { display: block; }
body[data-direction="d"] .mark-d { display: block; }

JS

// Tweak panel: cycle directions
document.querySelectorAll('[data-set-direction]')
  .forEach(b => b.addEventListener('click', () => {
    document.body.dataset.direction = b.dataset.setDirection;
  }));

Persistence

// Optional: remember the pick
localStorage.setItem(
  'mark-direction',
  document.body.dataset.direction
);
